CSS Hiding Techniques Demo

Explore different ways to hide elements with CSS and understand their impact on layout, interaction, and accessibility.

1. `display: none;`

Completely removes the element from the document flow. It takes up no space and is not accessible to screen readers.

This box is hidden with `display: none;`

When hidden, the button below will move up to fill the space.

This box is below the toggled box.

2. `visibility: hidden;`

Makes the element invisible, but it still occupies its original space in the document flow. It is accessible to screen readers.

This box is hidden with `visibility: hidden;`

When hidden, the button below will NOT move up.

This box is below the toggled box.

3. `opacity: 0;`

Makes the element completely transparent. It still occupies space and is clickable/focusable. Great for fade effects!

This box is hidden with `opacity: 0;` (try clicking it when invisible!)

When hidden, the button below will NOT move up. The box is still clickable!

This box is below the toggled box.

4. `position: absolute;` (Off-screen)

Removes the element from the normal document flow and moves it far off-screen. It takes up no space in the layout.

This box is hidden by moving off-screen.

When hidden, the button below will move up. The box is still technically in the DOM.

This box is below the toggled box.

5. `width: 0; height: 0;` (`overflow: hidden;`)

Collapses the element's dimensions to zero, effectively hiding its content. Great for collapsible sections.

This box is hidden by collapsing its dimensions.

When hidden, the box below will move up. This method allows for smooth expand/collapse animations.

This box is below the toggled box.

6. `.sr-only` (Screen Reader Only / Visually Hidden)

Visually hides content while keeping it fully accessible to screen readers. Crucial for accessibility.

This box contains text that is *visually* hidden but still accessible to screen readers.

This text is hidden visually but a screen reader would announce it. This is useful for providing context to assistive technologies without cluttering the visual design.

Try inspecting this element with developer tools or using a screen reader to confirm its presence.

This technique is not meant for toggling visibility, but for permanent visual hiding with accessibility in mind.
